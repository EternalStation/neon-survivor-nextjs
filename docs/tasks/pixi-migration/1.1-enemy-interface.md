# Task 1.1 — Enemy Interface Refactoring

## Status: Done

## Dependencies: None

## Goal
Break the monolithic `Enemy` interface (~100+ fields) into composable sub-objects.

## Current State
All fields are flat on one level in `Types.ts` (~1,177 lines total). `Enemy` is used in ~40+ files.

## Target Structure

```
types/
  enemy.ts — Enemy + sub-interfaces
```

### Sub-interfaces

- **EnemyCore** — id, x, y, size, hp, maxHp, spd, dead, type
- **EnemyVisual** — shape, shellStage, palette, fluxState, pulsePhase, rotationPhase
- **EnemyAI** — seen, spiralAngle, spiralRadius, reachedRange, wobblePhase, jitterX, jitterY
- **EnemyCombat** — lastAttack, lastHitTime, invincibleUntil, frozen, slowFactor, slowUntil
- **EnemyStatusEffects** — isInfected, infectedUntil, infectionDmg, bleedTimer, bleedDmg, burnStack
- **EnemyElite** (nullable) — isElite, eliteState, originalPalette, rarePhase, rareTimer, rareIntent, dashState, dashAngle, dashCooldown
- **EnemyBoss** (nullable) — boss, bossType, bossAttackPattern, boss-specific fields
- **EnemyWorm** (nullable) — wormId, wormRole, wormSegmentIndex, wormHistory, wormBurrowState, wormAIState
- **EnemyLegion** (nullable) — legionId, legionLeadId, legionSlot, legionShield, legionReady

### Composed Enemy

```typescript
interface Enemy extends EnemyCore {
  visual: EnemyVisual
  ai: EnemyAI
  combat: EnemyCombat
  status: EnemyStatusEffects
  elite: EnemyElite | null
  boss: EnemyBoss | null
  worm: EnemyWorm | null
  legion: EnemyLegion | null
}
```

## Migration Strategy (incremental)
1. Create new sub-interfaces alongside old fields
2. Add adapter functions `toNewEnemy(old)` / `toLegacyEnemy(new)`
3. Migrate file by file, remove old fields as each consumer is updated
4. Final cleanup: delete flat fields, remove adapters
