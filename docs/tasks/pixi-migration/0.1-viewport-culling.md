# Task 0.1 — Viewport Culling Utility

## Status: Not Started

## Dependencies: None

## Goal
Create a viewport culling utility and apply it in every renderer to skip off-screen entities.

## Expected Impact
+30-50% FPS in scenes with enemies spread across the map.

## Implementation

### 1. Create `packages/shared/logic/rendering/ViewportCulling.ts`

```typescript
export interface Viewport {
  x: number
  y: number
  halfW: number
  halfH: number
}

export function isVisible(ex: number, ey: number, radius: number, vp: Viewport): boolean {
  return (
    ex + radius > vp.x - vp.halfW &&
    ex - radius < vp.x + vp.halfW &&
    ey + radius > vp.y - vp.halfH &&
    ey - radius < vp.y + vp.halfH
  )
}
```

### 2. Apply `isVisible()` checks in every renderer

Before drawing each entity, check visibility. Files to update:
- `EnemyRenderer.ts`
- `ProjectileRenderer.ts`
- `EffectRenderer.ts`
- `BossRenderer.ts`
- `PlayerRenderer.ts`
- `MapRenderer.ts` (grid lines)

### 3. Compute viewport from camera state

Derive `Viewport` from the current camera position and canvas dimensions each frame.

## Acceptance Criteria
- [ ] `ViewportCulling.ts` created with `isVisible()` function
- [ ] All renderers call `isVisible()` before drawing entities
- [ ] Off-screen entities are not rendered
- [ ] No visual regression (entities appear/disappear correctly at screen edges)
- [ ] FPS improvement measurable with 200+ enemies spread across map
