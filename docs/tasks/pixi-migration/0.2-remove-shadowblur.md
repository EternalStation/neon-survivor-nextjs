# Task 0.2 — Remove shadowBlur from Hot Paths

## Status: Not Started

## Dependencies: None

## Goal
Replace `shadowBlur` (the single most expensive Canvas 2D operation) with pre-rendered glow sprites.

## Implementation

### 1. Create glow sprite generator

At startup, pre-render a set of glow circle images (8-10 sizes) using an offscreen canvas:
- Render radial gradients at various sizes (e.g. 16, 24, 32, 48, 64, 96, 128, 192 px)
- Store as `HTMLCanvasElement` or `ImageBitmap` for fast `drawImage()` calls

### 2. Replace shadowBlur in renderers

Files to modify:
- `EnemyRenderer.ts` — enemy glow effects
- `ProjectileRenderer.ts` — bullet glow
- `EffectRenderer.ts` — area effect glow
- `BossRenderer.ts` — boss aura
- `PlayerRenderer.ts` — player shield glow

### 3. Approach per file

Replace patterns like:
```typescript
ctx.shadowBlur = 20
ctx.shadowColor = color
ctx.fill()
ctx.shadowBlur = 0
```

With:
```typescript
ctx.drawImage(glowSprite[sizeIndex], x - halfSize, y - halfSize)
ctx.fill()
```

## Acceptance Criteria
- [ ] Glow sprite set generated at game startup
- [ ] All `shadowBlur` usage removed from per-entity rendering
- [ ] Visual quality is acceptable (glow effect preserved)
- [ ] Measurable FPS improvement (shadowBlur is 10-50x slower than drawImage)

## Files
- **New**: Glow sprite generator utility
- **Modified**: `EnemyRenderer.ts`, `ProjectileRenderer.ts`, `EffectRenderer.ts`, `BossRenderer.ts`, `PlayerRenderer.ts`
