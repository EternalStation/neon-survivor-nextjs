# Task 1.2 — Player Interface Refactoring

## Status: Done

## Dependencies: None

## Goal
Break the monolithic `Player` interface (~90 fields) into composable sub-objects.

## Target Structure

### Sub-interfaces

- **PlayerCore** — x, y, size, curHp, playerClass, level, xp
- **PlayerCombatStats** — hp, dmg, atk, crit, speed, reg, arm, pierce, multi, droneCount
- **PlayerTracking** — damageDealt, damageTaken, damageBlocked, damageBreakdown, incomingDamageBreakdown, healingBreakdown, avgHpAccumulator, avgHpSampleCount
- **PlayerMovement** — lastAngle, targetAngle, faceAngle, knockbackX, knockbackY
- **PlayerSkills** — activeSkills, lastShot, classShotCount, class-specific skill state
- **PlayerBuffs** — invincibleUntil, stunnedUntil, phaseShiftUntil, immobilized, healingDisabled, invertedControlsUntil
- **PlayerShields** — shield, shieldExpiry, shieldChunks

### Composed Player

```typescript
interface Player extends PlayerCore {
  stats: PlayerCombatStats
  tracking: PlayerTracking
  movement: PlayerMovement
  skills: PlayerSkills
  buffs: PlayerBuffs
  shields: PlayerShields
  inventory: (Upgrade | null)[]
  upgradesCollected: number
  rerolls: number
}
```
