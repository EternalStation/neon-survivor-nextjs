# Task 4.2 — SpatialGrid Numeric Keys

## Status: Not Started

## Dependencies: None (can be done independently)

## Goal
Optimize `SpatialGrid` by replacing string keys with numeric bit-packed keys to eliminate string allocation GC pressure.

## Current Implementation
Uses `Map<string, Enemy[]>` with string key concatenation — creates garbage every query.

## New Implementation

```typescript
class SpatialGrid {
  private cells: Map<number, Enemy[]> = new Map()

  private key(col: number, row: number): number {
    return ((col + 32768) << 16) | (row + 32768)
  }

  add(enemy: Enemy): void {
    const k = this.key(
      Math.floor(enemy.x / this.cellSize),
      Math.floor(enemy.y / this.cellSize)
    )
    const cell = this.cells.get(k)
    if (cell) cell.push(enemy)
    else this.cells.set(k, [enemy])
  }

  query(x: number, y: number, radius: number, result: Enemy[]): void {
    result.length = 0
    // same logic but numeric keys, reusable result array
  }
}
```

## Changes
- Numeric keys (bit-packed) instead of string concatenation — no GC pressure
- Pre-allocated result array passed as parameter — no allocation per query
- Cell arrays reused between frames via `clear()` that empties arrays instead of deleting
