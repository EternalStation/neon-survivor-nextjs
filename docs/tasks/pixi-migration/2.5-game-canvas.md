# Task 2.5 — GameCanvas Component Update

## Status: Not Started

## Dependencies: 2.1, all 2.3.x layers

## Goal
Update `GameCanvas.tsx` to use PixiJS instead of Canvas 2D context.

## Current Implementation
```typescript
// Creates <canvas> and passes 2D context to renderGame()
const ctx = canvas.getContext('2d')
renderGame(ctx, gameState, meteoriteImages, scaleFactor, language)
```

## New Implementation
```typescript
export function GameCanvas({ gameRef }: Props) {
  const canvasRef = useRef<HTMLCanvasElement>(null)
  const pixiAppRef = useRef<PixiApp | null>(null)

  useEffect(() => {
    if (!canvasRef.current) return
    const app = new PixiApp()
    app.init(canvasRef.current).then(() => {
      pixiAppRef.current = app
    })
    return () => { app.destroy() }
  }, [])

  // Game loop calls pixiAppRef.current.render(gameState)

  return <canvas ref={canvasRef} />
}
```

## Render Function Signature Change

**Before** (Canvas 2D):
```typescript
function renderGame(
  ctx: CanvasRenderingContext2D,
  state: GameState,
  meteoriteImages: Map<string, HTMLImageElement>,
  scaleFactor: number,
  language: string
): void
```

**After** (PixiJS):
```typescript
function renderGame(
  pixiApp: PixiApp,
  state: GameState,
  language: string
): void
```

- No `ctx` — PixiJS manages its own context
- No `meteoriteImages` — PixiJS asset loader handles textures
- No `scaleFactor` — camera zoom handled by PixiCamera
