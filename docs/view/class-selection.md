# Экран выбора класса

## Назначение
Перед началом игры игрок выбирает один из доступных классов персонажа. Экран отображает ключевые характеристики каждого класса и позволяет подтвердить выбор перед стартом сессии. Опционально включается режим Tutorial.

---

## Когда отображается
- Перед каждой новой игровой сессией.
- Доступен только до старта игры (в меню до выбора класса).

---

## Структура экрана

### Заголовок
- Название раздела: «Class Selection» или аналогичный.
- Переключатель Tutorial — включает/отключает обучающие подсказки в ходе игры.

### Список классов
Карточки отображаются для всех 5 классов в фиксированном порядке:
1. [Malware](../classes/malware.md) — `malware`
2. [Void](../classes/void-eventhorizon.md) — `eventhorizon`
3. [Ray](../classes/ray-stormstrike.md) — `stormstrike`
4. [Vortex](../classes/vortex-aigis.md) — `aigis`
5. [Hive-Mother](../classes/hive-mother.md) — `hivemother`

## Карточка класса

### Элементы карточки
| Элемент | Описание |
|---------|----------|
| Иконка | Изображение класса или цветной гексагон с цветом темы |
| Название | Короткое имя класса (например, «Malware») |
| Подзаголовок | Эпическое название (например, «THE GLITCHED SOVEREIGN») |
| Основная способность | Название способности (`capabilityName`) |
| Описание способности | Текст `capabilityDesc` |
| Бейджи модификаторов | Список статовых бонусов/штрафов |

### Бейджи модификаторов статов
Бейдж показывается только если для класса задан соответствующий модификатор:

| Бейдж | Условие показа | Пример значения |
|-------|---------------|-----------------|
| HP | `stats.hpMult` задан | +20% / −15% |
| SPD | `stats.spdMult` задан, класс ≠ Malware | +10% |
| DMG | `stats.dmgMult` задан, класс ≠ Malware | +40% |
| ATK | `stats.atkMult` задан, класс ≠ Malware | −20% |
| ARM | `stats.armMult` задан, класс ≠ Malware | +30% |
| XP | `stats.xpMult` задан, класс ≠ Malware | +15% |
| REG | `stats.regMult` задан, класс ≠ Malware | +50% |
| PIERCE | класс = Malware | +1 |

**Особое поведение для Malware**: показывается только бейдж `PIERCE +1` и `HP` (штраф).
Остальные бейджи для Malware скрыты, так как их значения являются частью механики, а не прямыми модификаторами характеристик.

## Взаимодействие и навигация

### Выбор класса мышью
- Наведение курсора на карточку — предварительный выбор (hover/highlight).
- Клик по карточке — подтверждение выбора класса.

### Выбор класса с клавиатуры
| Клавиша | Действие |
|---------|---------|
| `A` / `←` | Предыдущий класс |
| `D` / `→` | Следующий класс |
| `Space` / `Enter` | Старт игры с выбранным классом |

### Переключатель Tutorial
- Состояние сохраняется в переменной сессии `tutorialEnabled`.
- При включённом Tutorial в ходе игры отображаются обучающие подсказки.

---

## Результат выбора
- Устанавливается `player.playerClass = selectedClassId`.
- Применяются базовые модификаторы статов класса.
- Начинается игровая сессия.

---

## Допущения
- Порядок отображения карточек не настраивается (определён в коде).
- Нет ограничений на повторный выбор класса между сессиями.

## Открытые вопросы
- Сохраняется ли выбранный класс между сессиями (последний использованный)?
- Есть ли визуальное выделение для ранее выбранного класса?
